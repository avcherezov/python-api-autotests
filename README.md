## Автоматизация тестирования API

Тесты написаны с использованием:
- Python
- Pytest
- Allure
- Pydantic
- Faker
- HTTPX

### Описание

Цель проекта - автоматизировать тестирование сервера, проверяя различные функции приложения, чтобы обеспечить его стабильную и корректную работу.

В проекте отражены следующие практики:
- API клиенты для взаимодействия с эндпоинтами
- Pytest фикстуры для многократно используемых тестовых сценариев
- Pydantic модели для валидации данных
- Json schema для обеспечения корректности API контрактов
- Генерация данных для имитации реальных сценариев
- Подготовка Allure отчета по итогам тестирования
- Сбор покрытия на основе swagger документации

### Подготовка рабочей среды

Перейдите в свою рабочую директорию и выполните следующие команды:
```bash
git clone https://github.com/avcherezov/python-api-autotests.git
cd python-api-autotests
```

Создайте и активируйте виртуальное окружение:
```bash
python3 -m venv venv
source venv/bin/activate
```

Установите необходимые зависимости:
```bash
pip install -r requirements.txt
```

Запустите тесты:
```bash
python3 -m pytest
```

Просмотр отчёта Allure:
```bash
allure serve allure-results
```

Так же после прохождения тестов сгенерируется отчёт о покрытии тестового проекта, см. файл coverage.html
